It is a standard error handling strategy for network applications. The Tag Manager API is designed with the expectation that clients which choose to retry failed requests do so using exponential backoff. Besides being \"required\", using exponential backoff increases the efficiency of bandwidth usage, reduces the number of requests required to get a successful response, and maximizes the throughput of requests in concurrent environments.\n\nThe flow for implementing simple exponential backoff is as follows.\n\nMake a request to the API.\nReceive an error response that has an error code that can be retried.\nWait 1 second + random_number_milliseconds seconds.\nRetry request.\nReceive an error response that has an error code that can be retried.\nWait 2 seconds + random_number_milliseconds seconds.\nRetry request.\nReceive an error response that has an error code that can be retried.\nWait 4 seconds + random_number_milliseconds seconds.\nRetry request.\nReceive an error response that has an error code that can be retried.\nWait 8 seconds + random_number_milliseconds seconds.\nRetry request.\nReceive an error response that has an error code that can be retried.\nWait 16 seconds + random_number_milliseconds seconds.\nRetry request.\nIf you still get an error, stop and log the error.\n\nIn the above flow, random_number_milliseconds is a random number of milliseconds less than or equal to 1,000. This is necessary to avoid certain lock errors in some concurrent implementations. random_number_milliseconds must be redefined after each wait.\n\nNote: The wait is always (2 ^ n) + random_number_milliseconds, where n is a monotonically increasing integer initially defined as 0. The variable n is incremented by 1 for each iteration (each request).\n\nThe algorithm is set to terminate when n is 5. This ceiling is in place only to stop clients from retrying infinitely, and results in a total delay of around 32 seconds before a request is deemed \"an unrecoverable error.\"\n\nThe following Python code is an implementation of the above flow for recovering from errors that occur in a method called makeRequest.\n\nimport random\nimport time\nfrom apiclient.errors import HttpError\n\ndef makeRequestWithExponentialBackoff(tagmanager):\n \"\"\"Wrapper to request Google Tag Manager data with exponential backoff.\n\n The makeRequest method accepts the tagmanager service object, makes API\n requests and returns the response.
